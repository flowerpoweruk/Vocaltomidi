# Chord Palette

A Scaler-style chord-progression generator for Ableton Live, built on the
[Ableton Extensions SDK](https://ableton.github.io/extensions-sdk/). Right-click
a Session **clip slot** or a **MIDI clip** and choose **"Generate chord
progression…"** to open the picker.

![flow](https://img.shields.io/badge/Live-12.4.5%2B%20Suite%20Beta-orange)

## What it does

- **Key + scale picker** — 9 built-in scales (major, natural/harmonic/melodic
  minor, the four common modes, locrian), or pull the **current scale straight
  from Live** with one click.
- **Diatonic chord palette** — seven degree buttons (I…vii) that always show the
  correct in-key chord name for the selected chord type: triad, 7th, 9th, 6,
  sus2, sus4, add9.
- **Progression builder** — click chords to queue them, or load a **genre
  preset** (Pop, EDM Anthem, Future Bass, House, Emotional minor, Andalusian…).
- **Performance controls** — octave, chord length (beats), repeats, velocity,
  voicing (close / open / spread + bass), strum, and humanize.
- **One-click write** — generates the MIDI and writes it into the clip as a
  single undo step. Empty slots get a new clip sized to the progression.

Chords are generated by stacking **diatonic thirds within the chosen scale**, so
everything stays in key — the same idea behind commercial chord-suggestion
tools. The note generation lives in [`src/theory.ts`](src/theory.ts) (pure,
dependency-free); the SDK glue and UI live in
[`src/extension.ts`](src/extension.ts) and [`src/interface.html`](src/interface.html).

## Project layout

```
chord-palette/
├─ manifest.json        # Extension metadata read by Live
├─ build.ts             # esbuild bundle (inlines the HTML picker)
├─ src/
│  ├─ extension.ts      # activate(): context menu + command + write logic
│  ├─ theory.ts         # scales, chord stacking, note generation (pure)
│  ├─ interface.html    # the picker UI (Live dark theme)
│  └─ html.d.ts         # lets TS import the .html as a string
└─ vendor/              # SDK + CLI .tgz go here (gitignored — not redistributable)
```

## Build & run

> **Full step-by-step setup is in [INSTALL.md](INSTALL.md).** The short version:
>
> Requires the Ableton Extensions SDK distribution (Live 12 Suite Beta 12.4.5+)
> and Node.js ≥ 22.11 (the CLI requires ≥ 24.14).

1. Copy the two packages from your SDK download into `vendor/`:
   - `ableton-extensions-sdk-1.0.0-beta.0.tgz`
   - `ableton-extensions-cli-1.0.0-beta.0.tgz`

   (They are **not** committed to this repo — the SDK licence forbids
   redistributing it.)
2. Install and run:
   ```sh
   npm install
   npm run start        # builds and runs in Live's Extension Host
   ```
   `extensions-cli run` reads `EXTENSION_HOST_PATH` from your environment or a
   `.env` file (it points at Live's `ExtensionHostNodeModule.node`).
3. Package for distribution:
   ```sh
   npm run package      # produces a .ablx archive
   ```

## How it maps to the SDK

| Feature | SDK surface used |
| --- | --- |
| Context-menu entry | `ui.registerContextMenuAction("ClipSlot" \| "MidiClip", …)` |
| Resolve the clicked object | `getObjectFromHandle(handle, DataModelObject)` → `instanceof` |
| Read Live's current key/scale | `application.song.rootNote` / `scaleName` / `scaleIntervals` |
| Picker UI | `ui.showModalDialog(dataUrl, w, h)` (HTML posts `close_and_send`) |
| Create a clip in an empty slot | `ClipSlot.createMidiClip(lengthBeats)` |
| Write notes | `MidiClip.notes = NoteDescription[]` |
| Single undo step | `withinTransaction(() => { … })` |
